Extending to Multiple Input Features: House Price Prediction

Predicting Price from Size and Number of Bedrooms

1

🏠 Problem Setup & Data: House Price Prediction

Now, instead of just one feature, we'll use two features to predict house prices. Our model equation expands to include a weight for each feature:

ŷ = w₁x₁ + w₂x₂ + b

Where:

  • ŷ = predicted House Price
  • x₁ = House Size (sq ft)
  • x₂ = Number of Bedrooms
  • w₁ = weight for House Size
  • w₂ = weight for Number of Bedrooms
  • b = bias (intercept)

⚠️ Data Scaling (Normalization) is Essential!

When dealing with multiple features, especially if they have vastly different scales (like square footage vs. number of bedrooms), **normalization** is critical. We scale each feature (x₁ and x₂) and the target (y) to a common range (e.g., 0 to 1). Gradient Descent performs much more stably and efficiently with scaled data.

The calculations below and the training simulation will operate on these scaled values, and the final parameters will be the scaled weights (w₁, w₂) and bias (b).

2

📏 Cost Function (Mean Squared Error)

The Mean Squared Error (MSE) remains our chosen cost function, measuring the average squared difference between actual and predicted values:

J(w₁,w₂,b) = (1/2n) × Σ(yᵢ - ŷᵢ)²

🔍 Breaking Down the Cost Function

Step 1: Calculate prediction for each data point: ŷᵢ = w₁x₁ᵢ + w₂x₂ᵢ + b

Step 2: Find the error: (yᵢ - ŷᵢ)

Step 3: Square the error: (yᵢ - ŷᵢ)²

Step 4: Sum all squared errors and average: (1/2n) × Σ(yᵢ - ŷᵢ)²

Note: All calculations are done using scaled feature and target values.

3

∇ Partial Derivatives (Gradients)

To minimize J(w₁,w₂,b), we need to find the partial derivative with respect to each parameter (w₁, w₂, and b). Each derivative tells us how sensitive the cost is to a change in that specific parameter, holding others constant.

📐 Derivative with respect to Weight w₁:

∂J/∂w₁ = (1/n) × Σ(ŷᵢ - yᵢ) × x₁ᵢ

📐 Derivative with respect to Weight w₂:

∂J/∂w₂ = (1/n) × Σ(ŷᵢ - yᵢ) × x₂ᵢ

📐 Derivative with respect to Bias b:

∂J/∂b = (1/n) × Σ(ŷᵢ - yᵢ)

🧠 Intuitive Understanding

The gradients tell us the direction of steepest increase in cost. Gradient Descent moves parameters in the opposite direction (downhill) to minimize the cost. For instance, if ∂J/∂w₁ is positive, it means increasing w₁ would increase the cost, so we need to decrease w₁ to reduce the cost.

5

🚀 Learning Rate & Parameter Updates

🔄 Gradient Descent Algorithm for Multiple Linear Regression

Step 1: Initialize parameters w₁, w₂, and b randomly (or to 0).
Step 2: For each iteration, calculate predictions (ŷᵢ) and the cost J(w₁,w₂,b) using the current parameter values.
Step 3: Calculate the partial derivatives (gradients) ∂J/∂w₁, ∂J/∂w₂, and ∂J/∂b.
Step 4: Update each parameter simultaneously using the learning rate (α) and its respective gradient:
w₁new = w₁old - α × ∂J/∂w₁
w₂new = w₂old - α × ∂J/∂w₂
bnew = bold - α × ∂J/∂b
Step 5: Repeat steps 2-4 until convergence (cost change is minimal or max iterations reached).

🎛️ Learning Rate (α) Effects

The learning rate determines the size of the step taken down the cost function's gradient.
Too Small: Very slow convergence.
Too Large: May overshoot the minimum, oscillate, or even diverge.
